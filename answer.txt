Câu lệnh để cài đặt email và name là gì?
git config --global user.email "email"
git config --global user.name "name"

Cách tạo repo, push code lên repo
1. Tạo repo trên web GitHub
2. Tại terminal, nhập câu lệnh
git init -> khởi tạo thư mục được quản lý bởi git (tạo ra 3 vùng: working directory, staging, repository)
git commit -m"init project"
git add . -> đưa toàn bộ file từ vùng working directory vào vùng staging
git commit -m"init project" -> tạo 1 commit với nội dung bất kỳ
git remote add origin git@github.com:thanhhalethi/git-course.git
git push origin main

Push code lên git sau mỗi update
git add .
git commit -m "your commit message"
git push origin main

Cách tạo project playwright từ ban đầu
Tạo thư mục trên local
Mở thư mục đó trên terminal
Gõ câu lệnh npm init playwright@latest để khởi tạo playwright
Gõ câu lệnh code . để mở thư mục đó tại trình duyệt code

Git branching
- Xem branch đang có: git branch
- Tạo branh mới: git branch ten_branch
- Truy cập vào branch đã có: git checkout ten_branch
- Tạo branch mới và checkout luôn sang branch đó: git checkout -b ten_branch

Git merge
- Đầu tiên cần checkout sang nhánh muốn được merge vào (nhánh target)
git checkout main
- Gõ câu lệnh git merge + tên nhánh
git merge featureA => merge các file trong nhánh featureA vào nhánh main

Git - Thay đổi commit message sau khi đã commit
- Gõ câu lệnh git commit --amend
- Mặc định là chế độ Xem
- Để chuyển sang chế độ sửa, gõ "i" (insert)
- Thoát khỏi chế độ insert: nhấn ESC
- Vào chế độ dòng lệnh, gõ ":"
- Lưu và thoát: "wq"
- Hoặc chỉ cần gõ câu lệnh git commit --amend -m"your message here"

Git - git restore, git reset 
Có 3 vùng: working directory >> staging >> repository
- Đưa file từ vùng staging về vùng working directory
git restore --staged ten_file
- Đưa file từ vùng repository về vùng working directory
git reset HEAD~1
